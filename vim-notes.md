# Introduction
VIM is a configurable text editor that, when combined with a terminal shell, allows you to write, edit, and compile code or LaTeX all from your keyboard. You never even have to touch your mouse.

I am a bit of VIM n00b myself, so I'm creating these notes as a way to stay organized. Maybe you can find it useful too...

## The Basics

Most of the basic VIM commands are covered well by `vimtutor`, so I will not repeat them here. Before proceeding, you should run `vimtutor` and read it end to end. It will take around 30-60 minutes, but it's well worth the effort!

The best way to get used to VIM is by using VIM. You'll be surprised how natural it becomes if you just dedicate a day to using it in your workflow. While it may seem slow at first, soon it will become second nature, and that's where the real benefits begin to show.

## Configuration
Most guides leave out configuration for the end, but it is the main way you edit how VIM behaves so I'm going to frontload it here.


VIM's behavior is managed by the `.vimrc` file, typically located at `$HOME/.vimrc` (Unix machines) or ... (Windows machines)
